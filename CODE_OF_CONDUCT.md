# Code of Conduct — zeqsdk-docs

## Our pledge

`zeqsdk-docs` is the public source and live snapshot of the Zeq SDK documentation site at **zeq.dev/sdk/**. It exists so developers — in any of 11 languages — can learn, build on, and audit the Zeq OS Mathematical Framework.

We, as contributors, translators, maintainers and readers, pledge to make participation in this repository a harassment-free experience for everyone, regardless of age, body size, visible or invisible disability, ethnicity, sex characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, caste, color, religion, or sexual identity and orientation.

We pledge to act and interact in ways that contribute to an open, welcoming, diverse, inclusive and accurate documentation community.

## Why this matters here

Documentation shapes how the framework is understood. Poor conduct in a docs repo has outsized downstream effects:

- Translators are often volunteers and are among the most exposed to rude reviews.
- Bug reports against docs are often the first contact someone has with the project.
- Misinformation in docs propagates into every application built on top of the SDK.

We therefore hold conduct here to the same standard as conduct in the code repos of the framework.

## Expected behaviour

Examples of behaviour that contributes to a positive environment:

- Using welcoming and inclusive language across all 11 locales.
- Respecting translators' choices; translation is interpretive work, not transliteration.
- Gracefully accepting constructive criticism on technical accuracy, wording and examples.
- Focusing on what is best for readers of the docs, not on scoring points.
- Showing empathy toward people who are learning the framework for the first time.
- Citing sources and linking to the relevant framework references when making technical claims in docs.
- Reporting broken links, wrong code samples and outdated pages clearly and constructively.

## Unacceptable behaviour

Examples of unacceptable behaviour include:

- Sexualised language or imagery and unwelcome sexual attention or advances.
- Trolling, insulting or derogatory comments, and personal or political attacks.
- Public or private harassment, including targeting specific translators or reviewers.
- Publishing others' private information, such as a physical or email address, without their explicit permission.
- Deliberately introducing misleading documentation, broken code samples or inaccurate translations.
- Edits that misrepresent the framework constants (1.287 Hz HulyaPulse, 0.777 s Zeqond, KO42 Metric Tensioner, ZEQ Framework v1.287).
- Other conduct which could reasonably be considered inappropriate in a professional setting.

## Scope

This Code of Conduct applies within all project spaces — issues, pull requests, translation reviews, discussions, commits and any communication tied to this repository — and also when an individual is officially representing the project in public spaces.

## Enforcement

Instances of abusive, harassing or otherwise unacceptable behaviour may be reported by contacting the maintainers via **[zeq.dev/pricing](https://zeq.dev/pricing)** (support channel). All complaints will be reviewed and investigated promptly and fairly.

Maintainers are obligated to respect the privacy and security of the reporter of any incident.

## Enforcement guidelines

Maintainers will follow these guidelines in determining consequences for any action they deem in violation of this Code of Conduct:

1. **Correction** — a private, written warning, with clarity around the nature of the violation and an explanation of why the behaviour was inappropriate. A public apology may be requested.
2. **Warning** — a warning with consequences for continued behaviour. No interaction with the people involved, including unsolicited interaction with those enforcing the Code of Conduct, for a specified period of time. Violating these terms may lead to a temporary or permanent ban.
3. **Temporary ban** — a temporary ban from any sort of interaction or public communication with the community for a specified period of time. No public or private interaction with the people involved is allowed during this period.
4. **Permanent ban** — a permanent ban from any sort of public interaction within the project community.
