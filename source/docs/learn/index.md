# Learn

The conceptual track. Start here if you want to understand *why* the Zeq kernel works the way it does before you start calling it.

## Core concepts

- **[KO42](./concepts/ko42)** — the metric tensioner that keeps every computation phase-locked.
- **[HulyaPulse](./concepts/hulyapulse)** — the 1.287 Hz heartbeat the entire system is synchronised to.
- **[Zeqond](./concepts/zeqond)** — the 0.777 s computational second.
- **[Master Equation](./concepts/master-equation)** — how 1,576 operators compose into one number.
- **[ZeqProof](./concepts/zeqproof)** — the HMAC-SHA256 binding that makes every result verifiable.
- **[ZeqState](./concepts/zeqstate)** — the canonical result object.
- **[Seven-Step Protocol](./concepts/seven-step-protocol)** — the wizard the kernel runs on every call.
- **[Precision Bound](./concepts/precision-bound)** — why every result is guaranteed ≤0.1%.

When you're ready to call it, jump to the **[Quickstart](../build/quickstart/introduction)**.
