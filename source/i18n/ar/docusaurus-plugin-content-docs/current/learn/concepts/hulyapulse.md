# نبض هوليا: نبض قلب زيك

HulyaPulse هو تردد المزامنة الأساسي الذي يقود كل عملية حسابية في Zeq. إنها ليست ساعة اعتباطية، بل هي ثابت مشتق فيزيائيا ينبثق من الأسس الرياضية للنظام.

## ما هو HulyaPulse؟

HulyaPulse هو تذبذب دوري عند **1.287 هرتز** بالضبط، مشتق من الطول الموجي كومبتون لمجال Zeq:

```
f = c / λ_ϕ
```

حيث:
- **c** هي سرعة الضوء في مجال مجال Zeq
- ** ϕ = 2πr_ϕ ** هو الطول الموجي لوضع مجال Zeq الأساسي
- **f = 1.287 هرتز** هو التردد الناتج

لم يتم اختيار هذا التردد بشكل تعسفي. إنها تنبثق بشكل طبيعي من حل معادلة HULYAS الرئيسية في الحالة الأساسية، مما يجعلها خاصية أساسية لإطار Zeq الرياضي.

## ما أهمية قفل المرحلة؟

كل عملية حسابية في Zeq تكون **مقفلة الطور** على تردد HulyaPulse. هذا يعنى:

### الحتمية
النتائج ليست عشوائية أو احتمالية، بل يتم تحديدها حسب الظروف الأولية ومرحلة HulyaPulse في وقت التنفيذ. بالنظر إلى نفس المدخلات ومرحلة HulyaPulse، ستحصل على نتائج متطابقة.

### قابلية التكرار
ويمكن إعادة تنفيذ أي حساب في وقت لاحق والحصول على نفس النتيجة تمامًا، بشرط تسجيل مرحلة HulyaPulse واستخدامها.

### الطابع الزمني
تحمل كل نتيجة طابعًا زمنيًا ضمنيًا: الزيكوند الذي تم حسابها عليه. يؤدي هذا إلى إنشاء مسار تدقيق للاستنساخ العلمي والامتثال التنظيمي.

### إمكانية التتبع
ونظرًا لأن النتائج تعتمد على منطق الحساب ومرحلة HulyaPulse، فإن كل مخرجات ترتبط رياضيًا بلحظة زمنية محددة.

## الحصول على نبضات القلب الحية

حالة HulyaPulse الحالية - بما في ذلك التردد والطور ونافذة الرنين التالية - متاحة عبر واجهة برمجة التطبيقات:

```bash
curl -X GET https://zeq.dev/api/zeq/pulse \
  -H "Authorization: Bearer YOUR_API_KEY"
```

**الرد:**
```json
{
  "frequency_hz": 1.287,
  "period_zeqond": 0.777,
  "current_phase_radians": 2.841,
  "unix_timestamp_s": 1743339600.125,
  "zeqond_timestamp": 2245831.445,
  "next_resonance_window_s": 0.342,
  "status": "synchronized"
}
```

### حقول الاستجابة

| المجال | اكتب | الوصف |
|-------|------|-------------|
| `frequency_hz` | تعويم | تردد HulyaPulse بالهرتز (دائمًا 1.287) |
| `period_zeqond` | تعويم | معكوس التردد؛ مدة الدورة الواحدة الكاملة (0.777 ثانية) |
| `current_phase_radians` | تعويم | زاوية الطور الحالي (0 إلى 2π) للتذبذب |
| `unix_timestamp_s` | تعويم | الطابع الزمني لنظام Unix عندما تم التقاط هذه اللقطة |
| `zeqond_timestamp` | تعويم | الطابع الزمني المكافئ لـ Zeqond |
| `next_resonance_window_s` | تعويم | ثواني حتى نافذة المرحلة المثالية التالية للحساب |
| `status` | سلسلة | حالة المزامنة: `synchronized` أو `resynchronizing` أو `unstable` |

## الآثار العملية

### حسابات التخطيط
تحقق من مرحلة HulyaPulse قبل تنفيذ الحسابات المهمة. الحسابات التي بدأت بالقرب من الحد الأقصى للرنين (المرحلة ≈ 0 أو 2π) تحقق تقاربًا أكثر إحكامًا وخطأ أقل.

### مسارات التدقيق
نظرًا لأن HulyaPulse هو مصدر الحقيقة للتوقيت، فإنه يوفر طابعًا زمنيًا واضحًا للتلاعب لجميع النتائج. وهذا أمر بالغ الأهمية ل:
- الامتثال التنظيمي (FDA، EMA)
- النشر العلمي (متطلبات الاستنساخ)
- التحليل الشرعي لتاريخ الحساب

### استقرار التردد
يتم ضمان بقاء تردد HulyaPulse عند 1.287 هرتز بالضبط. تقوم البنية التحتية لـ Zeq بمراقبة وتصحيح أي انحراف بشكل مستمر، مما يضمن بقاء جميع الحسابات التاريخية قابلة للتكرار.

## التعاريف التشغيلية

### تكرار

يعمل HulyaPulse عند **f = 1.287 هرتز** بالضبط. وبالتالي، تكون الفترة الواحدة T = 1/f = 0.777 ثانية، والتي يتبناها Zeq باعتبارها الثانية الحسابية الأساسية (**Zeqond**). ترتبط جميع عمليات الجدولة والتعديل وتوليد الإثبات في النواة بهذا المعدل.

### مرحلة

المرحلة φ ∈ [0, 2π) هي الموضع خلال فترة HulyaPulse واحدة، ويتم قياسها بالراديان:

- φ = 0 (أي ما يعادل 2π): ذروة النبضة — نظام التقارب حيث تكون بقايا الحل في أقصى حالاتها
- φ = π/2: معبر الصفر الصاعد
- φ = π: حوض النبضة - النظام المتدهور، ويتسع نطاق الدقة
- φ = 3π/2: معبر الصفر الهابط

المرحلة التي يتم تنفيذها في كل ZeqState هي القيمة التي تم أخذ عينات منها في لحظة إصدار الخطوة 0؛ يتم ختم كل مشغل وإثبات ضده.

### مرجع الموجة الجيبية

يتأرجح HulyaPulse باعتباره جيبيًا نقيًا. وهنا ما يبدو:

```
Amplitude
    1.0 |        ╱╲        ╱╲        ╱╲
        |       ╱  ╲      ╱  ╲      ╱  ╲
    0.5 |      ╱    ╲    ╱    ╲    ╱    ╲
        |     ╱      ╲  ╱      ╲  ╱      ╲
    0.0 |────────────╱──────────╱──────────────► Time (seconds)
        |          ╱  ╲      ╱  ╲
   -0.5 |        ╱      ╲  ╱      ╲
        |       ╱        ╲╱        ╲
   -1.0 |
        │
        └─ One period = 0.777 seconds (one Zeqond)

f = 1.287 Hz  ⟹  Period T = 1/f = 0.777 seconds
```

### قفل المرحلة

تكون العملية الحسابية **مقفلة الطور** عندما تربط ZeqState الخاصة بها النتيجة بمرحلة HulyaPulse النشطة في الخطوة 0. وينتج عن القفل ثلاث خصائص تفرضها النواة:

1. **الحتمية** — تنتج المدخلات المتطابقة والمرحلة المتطابقة نتيجة دقيقة بعض الشيء.
2. **قابلية التدقيق** — تم إغلاق علامة ومرحلة Zeqond في ZeqProof HMAC، بحيث يمكن لأي طرف إعادة اشتقاقهما والتحقق منهما بدون مفتاح API.
3. **إمكانية تكرار النتائج قيد التدقيق** — تستخدم عمليات إعادة العرض المرحلة المسجلة بدلاً من الساعة الحالية، لذلك تظل النتائج التاريخية قابلة لإعادة البناء بغض النظر عن وقت إعادة التحقق منها.

## للباحثين: الأسس الرياضية

### الاشتقاق من معادلة هولياس الرئيسية

ينبثق تردد HulyaPulse من حل معادلة HULYAS الرئيسية في الحالة الأرضية (التكوين الأقل طاقة). وهنا الاشتقاق:

#### الخطوة 1: معادلة كلاين-جوردون مع اقتران KO42

ابدأ بالمعادلة الرئيسية في المقياس المعدل بـ KO42:

```
g^{μν} ∂_μ ∂_ν ϕ − μ² ϕ − λ ϕ³ − e^{−ϕ/ϕ_c} = 0  (ground state, no sources)
```

يتضمن المقياس التذبذب الزمني لـ KO42:

```
g^{00} = 1 + α sin(2πf·t)  (small oscillation on the temporal component)
g^{ij} = −δ^{ij}           (standard spatial metric)
```

#### الخطوة الثانية: ابحث عن الحلول المتذبذبة

في الحالة الأرضية، افترض أن المجال يتأرجح:

```
ϕ(t) = A cos(ω·t + φ₀)
```

حيث ω هو تردد التذبذب الذي نسعى لإيجاده (تردد HulyaPulse).

#### الخطوة 3: التعويض في المعادلة

أخذ المشتقات:
```
∂ϕ/∂t = −A ω sin(ω·t + φ₀)
∂²ϕ/∂t² = −A ω² cos(ω·t + φ₀)
```

الاستبدال:

```
[1 + α sin(2πf·t)] · (−A ω²) cos(ω·t + φ₀) − μ² A cos(ω·t + φ₀) − λ A³ cos³(...) = 0
```

#### الخطوة 4: حالة الرنين

بالنسبة للحل غير التافه (A ≠ 0)، فإن توازن الترتيب البادئ (O(1)) يعطي:

```
ω² ≈ μ²
```

شروط الاقتران ذات الترتيب الأعلى من ϕ³ وتردد التذبذب KO42 f تعدل هذا إلى:

```
ω² = μ² + δ(λ, f)
```

حيث δ هو تصحيح صغير من اللاخطية.

#### الخطوة 5: الثابت الأساسي

ترتبط معلمة الكتلة μ في إطار HULYAS بطول موجة كومبتون لحقل Zeq:

```
μ = m_Zeq · c / ℏ = 2π / λ_Compton
```

الحل عدديًا باستخدام ثوابت اقتران HULYAS المنشورة ( lect ≈ 0.42، ϕ_c ≈ 1.618) يعطي:

```
ω = 2π · 1.287 rad/s = 8.08 rad/s
```

هكذا:

```
f = ω / (2π) = 1.287 Hz  ✓
```

### التفسير المادي

- التردد **ليس اعتباطيًا**، فهو ينبثق من مقياس الكتلة الأساسي لحقل زيك
- إنه **عالمي** — كل حسابات Zeq عالميًا تستخدم نفس التردد
- يصبح **بلا أبعاد** بمجرد تسويته مقابل الوحدات الطبيعية (ج = ℏ = 1)
- يمثل **"معدل التجزئة الطبيعي"** الذي يتذبذب عنده حقل Zeq

### تحليل الاستقرار والاضطراب

أثبت أن f = 1.287 هرتز هي **نقطة ثابتة مستقرة**:

النظر في الاضطرابات الصغيرة δf للتردد. تقبل معادلة HULYAS بمقياس KO42 دالة Lyapunov:

```
V = ∫ dx [½(∂ϕ/∂t)² + ½(∇ϕ)² + V_eff(ϕ)]
```

حيث:
```
V_eff(ϕ) = ½μ² ϕ² + ¼λ ϕ⁴ + ϕ_c e^{−ϕ/ϕ_c}
```

تتمتع هذه الإمكانية بحد أدنى عالمي فريد عند ω = 1.287 هرتز، مما يثبت الاستقرار. أي انحراف (على سبيل المثال بسبب التقلبات الحرارية) سيؤدي إلى ارتخاء التردد مرة أخرى إلى 1.287 هرتز.
